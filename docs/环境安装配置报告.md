#### 5.2 SQLAlchemy保留字段冲突修复

**问题**: `metadata`是SQLAlchemy的保留属性名

**修复文件**:
- `backend/models/case.py:38`
- `backend/models/log_file.py:57`
- `backend/models/diagnosis.py:51`

**修复方式**:
```python
# 修复前
metadata = Column(JSON, default=dict)
'metadata': self.metadata

# 修复后
meta_data = Column('metadata', JSON, default=dict)
'metadata': self.meta_data
```

#### 5.3 异步引擎连接池配置修复

**问题**: 异步引擎不能使用`QueuePool`

**修复文件**: `backend/database.py:73`

**修复方式**:
```python
# 修复前
self._async_engine = create_async_engine(
    settings.DATABASE_URL_ASYNC,
    poolclass=QueuePool,  # ❌ 错误
    ...
)

# 修复后
self._async_engine = create_async_engine(
    settings.DATABASE_URL_ASYNC,
    # 不指定poolclass,让SQLAlchemy自动选择AsyncAdaptedQueuePool
    ...
)
```

#### 5.4 Parser Registry导出修复

**问题**: `worker.py`导入`parser_registry`但`__init__.py`只导出`registry`

**修复文件**: `backend/services/parser/__init__.py`

**修复方式**:
```python
from .base import BaseParser, ParsedEvent, ParserRegistry, registry

# 为了兼容性，提供parser_registry别名

```

## 🔧 遇到的问题及解决方案

### 问题1: ModuleNotFoundError: No module named 'backend'

**原因**: 代码中使用了绝对导入路径`from backend.xxx`，但运行时Python路径不包含backend的父目录

**解决方案**: 将所有`from backend.xxx`改为相对导入`from xxx`

**影响文件**: database.py, db_operations.py, test_database.py

---

### 问题2: sqlalchemy.exc.InvalidRequestError: Attribute name 'metadata' is reserved

**原因**: SQLAlchemy的Base类已经使用`metadata`作为类属性，不能在模型中定义同名字段

**解决方案**: 
- 将Python属性名改为`meta_data`
- 使用Column的第一个参数指定数据库列名为`'metadata'`
- 更新`to_dict()`方法使用`self.meta_data`

**影响文件**: case.py, log_file.py, diagnosis.py

---

### 问题3: sqlalchemy.exc.ArgumentError: Pool class QueuePool cannot be used with asyncio engine

**原因**: 异步引擎需要使用异步兼容的连接池类

**解决方案**: 移除`poolclass=QueuePool`参数，让SQLAlchemy自动选择`AsyncAdaptedQueuePool`

**影响文件**: database.py

---

### 问题4: ModuleNotFoundError: No module named 'base'

**原因**: parser模块中使用了错误的导入`from base import`，应该使用相对导入

**解决方案**: 批量修改为`from .base import`

**影响文件**: 所有parser_*.py文件(7个)

---

### 问题5: ImportError: cannot import name 'parser_registry'

**原因**: `worker.py`导入`parser_registry`但`__init__.py`只导出`registry`

**解决方案**: 在`__init__.py`中添加别名`parser_registry = registry`

**影响文件**: services/parser/__init__.py

---

### 问题6: PermissionError: [Errno 13] Permission denied: '/var/fota'

**原因**: 普通用户没有权限在/var下创建目录

**解决方案**: 使用sudo创建目录并修改所有者为deploy用户

```bash
sudo mkdir -p /var/fota/logs
sudo chown -R deploy:deploy /var/fota
```
