### 2. Parser Service基础框架 ✅

**目录**: [`backend/services/parser/`](backend/services/parser/)

#### 2.1 核心组件

**[`base.py`](backend/services/parser/base.py)** - 基础类和接口定义
- `ParsedEvent`: 标准化事件数据模型
- `BaseParser`: 抽象基类，定义解析器接口
- `ParserRegistry`: 解析器注册表，支持插件化架构
- `EventLevel` / `EventType`: 事件级别和类型枚举

**关键设计原则**:
1. **流式处理**: 逐行解析，避免内存溢出
2. **时间窗口裁剪**: 支持快速通道优化（仅解析目标时间窗口）
3. **容错性**: 单行解析失败不影响其他行
4. **可扩展性**: 插件化架构，易于添加新解析器

#### 2.2 已实现的解析器

**[`parser_android.py`](backend/services/parser/parser_android.py)** - Android Logcat解析器
- 支持threadtime和time两种格式
- 提取时间戳、进程ID、线程ID、日志级别、标签
- 自动识别模块名
- 正则表达式高效解析

**[`parser_fota.py`](backend/services/parser/parser_fota.py)** - FOTA文本日志解析器
- 识别FOTA升级阶段（init/check/download/verify/install/reboot/complete/failed）
- 提取关键信息（版本号、文件名、错误码、进度百分比）
- 自动分类事件类型（FOTA_STAGE/ERROR/WARNING/STATE_CHANGE）
- 结构化字段存储

**特性**:
- ✅ 时间窗口过滤（快速通道优化）
- ✅ 最大行数限制
- ✅ 解析置信度标记
- ✅ 原始日志回溯支持

### 2. Time Alignment Service（预计2-3天）

**文件**: 需创建 `backend/services/time_alignment.py`

**核心功能**:
- 锚点事件识别（跨域时间同步基准）
- Offset拟合算法（计算时钟偏移量）
- `normalized_ts`生成（统一时间轴）
- 三级降级策略：
  - 全部域对齐成功 → 正常分析
  - 部分域对齐失败 → 继续分析+警告标签
  - 全部域对齐失败 → 降级模式+醒目警告

**技术要点**:
- 多域日志时间同步算法
- 时钟漂移补偿
- 异常时间戳检测
- 置信度量化（0.0-1.0）
