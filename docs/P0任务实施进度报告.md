### 2. Parser Service基础框架 ✅

**目录**: [`backend/services/parser/`](backend/services/parser/)

#### 2.1 核心组件

**[`base.py`](backend/services/parser/base.py)** - 基础类和接口定义
- `ParsedEvent`: 标准化事件数据模型
- `BaseParser`: 抽象基类，定义解析器接口
- `ParserRegistry`: 解析器注册表，支持插件化架构
- `EventLevel` / `EventType`: 事件级别和类型枚举

**关键设计原则**:
1. **流式处理**: 逐行解析，避免内存溢出
2. **时间窗口裁剪**: 支持快速通道优化（仅解析目标时间窗口）
3. **容错性**: 单行解析失败不影响其他行
4. **可扩展性**: 插件化架构，易于添加新解析器

#### 2.2 已实现的解析器

**[`parser_android.py`](backend/services/parser/parser_android.py)** - Android Logcat解析器
- 支持threadtime和time两种格式
- 提取时间戳、进程ID、线程ID、日志级别、标签
- 自动识别模块名
- 正则表达式高效解析

**[`parser_fota.py`](backend/services/parser/parser_fota.py)** - FOTA文本日志解析器
- 识别FOTA升级阶段（init/check/download/verify/install/reboot/complete/failed）
- 提取关键信息（版本号、文件名、错误码、进度百分比）
- 自动分类事件类型（FOTA_STAGE/ERROR/WARNING/STATE_CHANGE）
- 结构化字段存储

**特性**:
- ✅ 时间窗口过滤（快速通道优化）
- ✅ 最大行数限制
- ✅ 解析置信度标记
- ✅ 原始日志回溯支持

### 4. Time Alignment Service ✅

**文件**: [`backend/services/time_alignment.py`](backend/services/time_alignment.py)

**核心功能**:
- **锚点事件识别**: 在多个日志源中找到同一物理事件的时间戳作为对齐基准
- **Offset拟合**: 为每个日志源计算时钟偏移量和可信度
- **normalized_ts生成**: 所有事件统一转换为绝对时间戳
- **三级降级策略**: SUCCESS/PARTIAL/FAILED，确保诊断流程不被阻断

**关键特性**:
- 支持5种锚点事件类型（system_boot/network_connected/fota_download_start/fota_install_start/system_reboot）
- 加权平均算法计算时钟偏移
- 置信度评分（样本数量 × 锚点置信度 × 一致性因子）
- 自动降级策略，避免时间对齐失败阻断诊断

### 6. 新增5个日志解析器 ✅

**文件**:
- [`backend/services/parser/parser_kernel.py`](backend/services/parser/parser_kernel.py)
- [`backend/services/parser/parser_mcu.py`](backend/services/parser/parser_mcu.py)
- [`backend/services/parser/parser_dlt.py`](backend/services/parser/parser_dlt.py)
- [`backend/services/parser/parser_ibdu.py`](backend/services/parser/parser_ibdu.py)
- [`backend/services/parser/parser_vehicle_signal.py`](backend/services/parser/parser_vehicle_signal.py)

#### 6.1 KernelParser - Kernel日志解析器
- 支持kernel log、tombstone、ANR三种格式
- 识别kernel panic、crash signal、backtrace
- 提取进程ID、线程ID、信号类型
- 自动识别kernel子系统模块

#### 6.2 MCUParser - MCU日志解析器
- 支持3种MCU日志格式变体
- 处理相对时间戳（uptime）
- 自动推断日志级别和事件类型
- 识别FOTA、CAN、电源等事件

#### 6.3 DLTParser - AUTOSAR DLT解析器
- 支持完整DLT格式和简化格式
- 提取ECU ID、Application ID、Context ID
- 解析消息类型（log/app_trace/nw_trace/control）
- 支持verbose和non-verbose模式

#### 6.4 IBDUParser - iBDU电源管理解析器
- 智能提取电压、电流、温度、百分比
- 自动识别电源相关事件
- 支持多种时间戳格式
- 电气参数异常检测

#### 6.5 VehicleSignalParser - 车辆信号解析器
- 支持CSV和表格两种格式
- 解析车速、转速、档位等关键信号
- 信号分类（speed/power/engine/transmission/brake/steering）
- 异常值检测（电压/温度/速度）
